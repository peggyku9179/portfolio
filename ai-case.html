02 / 08 · 專案背景

Project
Background

本專案為某國際汽車輪胎品牌之 AI Agent 設定案例，目標是協助經銷商第一線人員快速取得產品資訊，並降低短影音內容製作門檻。

專案需求涵蓋 AI 客服角色設定、品牌知識庫建置，以及針對短影音腳本生成的 Agent 功能模組設計。

說明

因專案涉及品牌內部資料，作品集中已去除品牌名稱、內部連結、活動細節與完整 Prompt 原文，僅呈現方法架構與優化流程。

測試範例

Agent
Architecture

我將 AI 設定拆成可維護的模組，讓後續團隊能理解每一段設定的用途，而不是只拿到一大段 Prompt。

每個模組獨立可調整，當品牌資料更新時，只需修改對應層級，不影響整體架構的穩定性。

06 / 08 · 測試與優化

QA Testing

測試情境 優化重點
產品名稱不完整 AI 需先追問確認，不可猜測
詢問產品特色 優先回答產品資訊，不引導至競品比較
詢問活動規則 資料不足時不可自行補充，需說明待確認
需要短影音腳本 提供分鏡、口播與拍攝提醒，格式標準化
語氣太像客服 調整為第一線人員可接受的口吻，降低疏離感

07 / 08 · Before / After

優化案例

Before

「這款產品是專門為電動車開發。」

⚠ 未確認資料，容易造成產品定位誤導

After

「若公開資料或品牌核准內容沒有明確標示電動車定位，AI 不應直接宣稱，需改以較保守且可查證的產品描述。」

此案例已去識別化，僅呈現優化邏輯

Reflection

反思

這次經驗讓我理解，AI Agent 建置不是單純寫 Prompt，而是需要結合資料整理、使用者情境設計、品牌語氣控管、測試驗證與持續優化的完整流程。
